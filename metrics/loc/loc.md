# 架构度量指标规范：代码行数 (Lines of Code - LOC)

## 1. 指标定义

**代码行数 (Lines of Code, LOC)** 是软件工程中最基础、最直观的规模度量指标。在 `arch-lens` 静态分析引擎中，LOC 细分为：

* **原始代码行 (Raw LOC)**：源文件中的绝对总行数（包含代码、空行和注释，对应文件的物理文本行数）。
* **逻辑代码行 (Logical LOC)**：去除干扰信息后的实际有效代码行数（**仅排除纯空行和纯注释行**）。

在“上帝类”或“上帝文件”的判定中，LOC 虽不单独作为决定性因子，但常作为**体积的基础触发阈值**（如：类 $Logical\ LOC > 800$ 或文件 $Raw\ LOC > 1500$）。

---

## 2. 核心价值与局限性

* **价值**：能够快速筛选出系统中体量异常的“巨型资产”，作为架构治理扫描的第一道粗筛漏斗。
* **局限性**：无法反映代码的真实内聚性与复杂度。一个包含 2000 行简单 `switch-case` 或纯数据结构定义的类，其架构坏味道可能远低于一个只有 150 行但包含多层嵌套、严重违反 SRP 原则的类。

---

## 3. 计算方式与规则细节

在 `arch-lens` 引擎中，为了确保多语言（Go/Java/C++）检测的通用性与性能，LOC 的扫描规则定义如下：

### 3.1 计入 LOC 的元素

1. **所有包含有效可执行代码的行**：如赋值、方法声明、控制流头部（`if`、`for`）、`return` 语句等。
2. **单花括号行**：在 C-Style 语言中，单独占一行的 `{` 或 `}`，由于其属于代码结构的一部分（非空行且非注释），**保留计入 LOC**。

### 3.2 排除在 LOC 之外的元素

1. **纯空白行**：不包含任何文本，或仅包含制表符/空格的行。
2. **纯注释行**：以单行注释开头（如 `//`、`#`）或完全处于多行注释块（`/* ... */`）内部的物理行。*（注：若代码与注释混在同一行，如 `int a = 1; // 注释`，仍按有效代码行计入 LOC）*

---

## 4. 典型误判场景与约束说明 (Edge Cases)

为防止大文本、自动生成代码导致误报，静态分析引擎必须应用以下约束：

* **约束 A（大字段定义）**：若类中包含大数组初始化、大型 JSON/XML 文本常量嵌在代码中，会导致 LOC 激增。引擎应支持设置单条语句最大行数权重，或识别常量块进行行数扣除。
* **约束 B（特化自动生成）**：由 Protocol Buffers、Thrift 或 ORM 框架自动生成的代码文件，其 LOC 往往成千上万，必须通过文件头特征（如 `// Code generated by...`）直接忽略。
* **约束 C（代码格式化风格影响）**：
  由于不同的开发人员或团队可能采用不同的代码格式化工具（例如有的习惯将花括号换行，有的习惯不换行），这会导致 LOC 产生小范围上下浮动。架构治理团队应在项目中强制推行统一的格式化规范（如 Go 使用 `gofmt`，Java 使用固定的 `Formatter`），以消减此类环境与风格差异造成的指标波动。

---

## 5. 详细代码计算实例

以下是一段经典的 Java 业务代码，我们来分别计算它的 **Raw LOC（原始代码行）** 和 **LOC（逻辑代码行）**：

```java
package com.archlens.demo; // 行 1 - 有效代码

import java.util.List;     // 行 2 - 有效代码
                           // 行 3 - 纯空行
/**
 * 订单处理核心类      
 * 用于演示 LOC 计算        
 */                        // 行 4~7 - 纯注释行
public class OrderService {// 行 8 - 有效代码
    // 成员变量             // 行 9 - 纯注释行
    private List<String> cache; // 行 10 - 有效代码

    public void process(int status) { // 行 11 - 有效代码
        /* 过滤无效状态 */             // 行 12 - 纯注释行
        if (status < 0) {             // 行 13 - 有效代码
            return;                   // 行 14 - 有效代码
        }                             // 行 15 - 单独括号行（计入）

        System.out.println("Processing..."); // 行 16 - 有效代码
    }                                        // 行 17 - 单独括号行（计入）
}                                            // 行 18 - 单独括号行（计入）

```

### 计算结果对撞表：

| 行号 | 文本内容分类 | 是否计入 Raw LOC | 是否计入 LOC | 说明 |
| --- | --- | --- | --- | --- |
| **1 ~ 2** | Package & Import | **Yes** | **Yes** | 属于有效代码行 |
| **3** | 纯空行 | **Yes** | **No** | 纯空行过滤 |
| **4 ~ 7** | 多行注释 | **Yes** | **No** | 纯注释行过滤 |
| **8** | 类声明头部 | **Yes** | **Yes** | 属于有效代码行 |
| **9** | 单行注释 | **Yes** | **No** | 纯注释行过滤 |
| **10** | 成员变量声明 | **Yes** | **Yes** | 属于有效代码行 |
| **11** | 方法声明头部 | **Yes** | **Yes** | 属于有效代码行 |
| **12** | 单行注释块 | **Yes** | **No** | 纯注释行过滤 |
| **13 ~ 14** | `if` 条件与 `return` | **Yes** | **Yes** | 属于有效代码行 |
| **15** | 单独右花括号 `}` | **Yes** | **Yes** | 非空行非注释，保留计入 |
| **16** | 打印语句 | **Yes** | **Yes** | 属于有效代码行 |
| **17 ~ 18** | 结尾花括号 `}` | **Yes** | **Yes** | 非空行非注释，保留计入 |

* **最终原始代码行数 (Raw LOC)** = **18 行**
* **最终逻辑代码行数 (LOC)** = **11 行**（排除了第 3、4、5、6、7、9、12 行共 7 行空行与注释）
